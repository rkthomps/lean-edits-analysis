

## Definitions
- **Programming Session**: A programming session is defined as all of the work done on one GitHub commit. 

### Notes
#### Programming Session
Conceptually, a session refers to some unit of work.
Github commits are a nice unit to consider. 
If used as intended, Github commits capture units of work as the programmer defines them.  

## What do we want to know about a Programming Session 
- Which files were modified? How many edits were made in each modified file?
- Was there a change in the set of top-level declarations in the project?  
  Which declarations were added, which declarations were removed? 
  - It may be the case that this set is very large for a small reason (e.g. a namespace was added or deleted).
- Which declarations were changed?
